# Publication Targets

## Purpose

Use this file as a tracker, not as a source of invented deadlines. Fill in venue names, deadlines, and formatting constraints only after confirming them from the venue's official call for papers or submission page.

## Suggested Target Types

| Target type | Why it fits the current project state | Recommended priority |
|---|---|---|
| Applied cybersecurity conference | Good fit for hybrid architecture plus validation evidence | High |
| Security workshop | Strong fit if the emphasis is practical validation and early-stage results | High |
| Student research symposium or poster | Good showcase path while the project is still maturing | High |
| Journal submission | Better after the related-work section, longer soak evidence, and broader benign review are stronger | Medium |
| Product demo track | Useful only if the demo package and public evidence bundle are polished | Medium |

## Venue Evaluation Checklist

- Does the venue accept practical validation or systems papers, not only theoretical ML novelty?
- Is the audience likely to value evidence retention, lab methodology, and operator workflows?
- Would the current limitations still be acceptable for this venue?
- Does the format support a demo, poster, short paper, or full paper?
- Can the current results be presented honestly without adding unverified claims?

## Recommended First Publication Direction

Given the current repo state, the best first targets are likely to be:

- applied cybersecurity venues
- workshops that value operational validation
- student research symposium or poster formats

These formats align well with the current strengths:

- hybrid architecture
- repeatable lab scenarios
- prepared-environment validation
- clear limitations and future work

They are also more forgiving than a mature journal or product-track submission if the full-duration soak and broader benign review are still in progress.
