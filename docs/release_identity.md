## Naming Guidance

### Repo name guidance

- Keep the name descriptive and stable.
- Favor terms that match the current reality:
  - hybrid
  - evidence-driven
  - workspace
  - lab-validated
- Avoid names that imply:
  - enterprise production readiness
  - autonomous response
  - zero-false-positive performance
  - quantum capability

### Future website guidance

- Use a simple URL-friendly form such as `universalnids`, `universal-nids`, or `hybrid-nids-workspace`.
- Keep the landing page positioning aligned to the current state:
  - research-first
  - open-source-first
  - evidence-backed
- If a site is created before full deployment signoff, include a visible readiness note.

## Positioning Options

### Research-First

Best when the goal is publication, thesis use, conference demos, and professor or reviewer credibility.

Strengths:

- matches the current evidence base
- fits the current readiness language
- makes honest limitations easier to communicate
- keeps the paper and portfolio narrative aligned

Tradeoffs:

- sounds less commercial
- requires disciplined language around future product ambitions

### Open-Source-First

Best when the goal is community visibility, contributions, and portfolio discoverability.

Strengths:

- attractive to engineers and recruiters
- easy to pair with transparent roadmaps and evidence bundles
- supports future training or consulting offers

Tradeoffs:

- still needs a clear readiness statement
- can create support expectations if the repo looks too product-like

### Product-First

Best only when packaging, support, deployment boundaries, and live operational evidence are much stronger.

Strengths:

- clearer commercial story later
- easier to map to pricing and roadmap tiers

Tradeoffs:

- current repo state does not support this positioning cleanly
- risks overclaiming readiness
- invites questions about support, SLA, hot reload, and deployment hardening that are not yet fully answered

## Recommendation

Recommended initial positioning: `research-first` with an `open-source-first` presentation layer.

Why:

- the repo already has strong evidence for testing, lab execution, and prepared-environment validation
- the current verdict is controlled pre-deployment review, not production deployment
- this positioning helps with papers, portfolios, interviews, and thesis use without forcing premature product claims

## Best Initial Recommendation

- Best project name: `Universal NIDS`
- Best tagline: `Hybrid intrusion detection with evidence-backed lab validation.`

This pairing is simple, credible, and consistent with the repository's current scope.
