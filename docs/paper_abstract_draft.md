# Paper Abstract Draft

## Option 1: Academic Tone

This work presents a hybrid network intrusion detection workspace that integrates live packet capture, offline replay, optional log-adapter ingest, artifact static analysis, signature matching, statistical anomaly scoring, supervised machine learning, optional unsupervised anomaly scoring, and fusion-based alert adjudication inside a single repository-backed workflow. The main contribution is not a new detector in isolation, but an evidence-driven validation approach that connects software testing, repeatable lab scenarios, prepared-environment execution, and analyst review. In the current project state, the repository records `152` collected tests, a default result of `144` passed with `8` deselected, and `79.16%` coverage under a `72%` enforced floor. Validation evidence also includes `5` latest passing offline lab scenarios and `10` latest passing prepared-environment scenarios across `17` retained manifests. Prepared-environment results include live queue-loss accounting, a tuned benign-soak rerun that reduced the exercised sample to `0` alerts across `1416` flows, and restart-based operator workflows for rule refresh, model swap, and configuration override. The current evidence supports a controlled pre-deployment review posture. However, broader benign-traffic adjudication, a full-duration `6` to `12` hour soak, and any requirement for zero-downtime reload remain open. The paper argues for evaluating practical IDS systems through retained operational evidence rather than benchmark metrics alone.

## Option 2: Practitioner / Research Hybrid Tone

Intrusion detection projects often report promising model metrics while leaving unanswered the practical question of whether the system behaves well under realistic operator workflows and prepared live environments. This paper documents a hybrid NIDS workspace designed to reduce that gap. The system combines live capture, offline PCAP replay, optional Suricata and Zeek ingest, signature and anomaly detection, a supervised ensemble, an optional unsupervised path, fusion-based alerting, persistent evidence storage, and a dashboard/reporting layer. Rather than presenting the project as production-ready, we evaluate it through layered evidence: `152` collected tests with `144` passing in the default suite, `79.16%` coverage under CI enforcement, five latest passing offline scenario bundles, and ten latest passing prepared-environment scenarios across seventeen retained manifests. We further show queue-pressure and packet-loss accounting, analyst-driven false-positive tuning that removed the exercised benign-soak alerts, and restart-based validation for rule, model, and configuration maintenance workflows. These results show that the project is beyond prototype-only validation and suitable for controlled pre-deployment review. At the same time, the evidence remains bounded: the extended soak is still a `900` second pilot toward a `21600` second target, broader benign corpora are still needed, and hot reload has not been validated.

## Option 3: Concise Conference-Submission Tone

We present a hybrid intrusion detection workspace that unifies live capture, offline replay, optional adapter ingest, artifact static analysis, multi-engine detection, fusion, storage, and operator-facing reporting. The contribution is an evidence-driven validation framework that links code-level testing to repeatable offline scenarios and prepared-environment execution. The current repository records `152` collected tests, `144` passing default tests, and `79.16%` coverage under a `72%` floor. Validation includes `5` latest passing offline scenario runs and `10` latest passing prepared-environment scenarios across `17` retained manifests. Prepared-environment evidence covers queue-loss accounting, tuned benign-soak adjudication, restart recovery, and restart-based rule, model, and configuration workflows. The current verdict is a controlled pre-deployment candidate rather than a production-ready system. Remaining gaps include a full-duration soak beyond the current `900s` pilot, broader benign false-positive review, and any zero-downtime reload requirement.
